# 2026-7-20问题整理

*这是这个项目里难得的hgc（human generated content）了*

## UI

1. 组进度条内容应当改为该组的`本轮次使用/group_usage_limit`，因为很明显的，组的生命周期只持续到本轮次末，和组内key用量统计有相关性但是意义上存在偏差，同时组的总体统计信息应当环绕这个被重构
2. 组总体统计信息展示中的`轮次`应当改为`本轮次`以避免语义模糊
3. 上方的总信息展示面板中的`可用key`标注错误，应当使用状态为`Ready`的key的数量，而不是当前在用的key的数量，这会使得`可用key`永远小于等于组大小

## 功能

1. 当前`research`端点返回事件是从上游收到sse事件则返回，但是实际上频率特别低，据我观察甚至只有整个`research`任务完成之后才会返回，而下游的mcp客户端经常因此超时，应当改为固定时间拉取进度并向mcp客户端报告进度
2. `astrbot`中调用`research`端点依然报错`error: Tool tavily_research has an output schema but did not return structured content`，之前别的端点报过修复的时候可以参考之前的文档
