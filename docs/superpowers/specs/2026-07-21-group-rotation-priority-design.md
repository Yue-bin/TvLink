# 分组轮换优先级设计

## 目标

分组重建后，轮换从总剩余额度最高的组开始，避免分组求解器的内部桶顺序决定请求调度顺序。

## 范围

- 保持现有分组算法、分组大小和组内 Key 的加权选择不变。
- 在 `optimalGroups` 返回组状态后，为组间轮换建立确定顺序：总剩余额度降序。
- 两组总余量相同时，按组内 Key 名称的字典序比较，保证重建结果可复现。
- 重建后的首个活动组仍为索引 0；排序后该索引即余量最高组。

## 验证

在 `internal/pool/pool_test.go` 增加回归测试，构造分组总余量不同的 Key 集合，断言 `RebuildGroups` 后 `groups` 按剩余额度降序排列，且首次 `Select` 使用第一组。
